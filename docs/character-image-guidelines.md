# キャラクター画像ガイドライン(受け入れ仕様)

すぴたろうの実画像を将来安全に配置するための仕様と手順。
現時点で実画像は未登録であり、UI は「画像準備中」プレースホルダーを表示する。
**架空の画像・AI 生成画像・第三者素材を置いてはいけない。**

## 推奨ファイル形式

1. 第一候補: **透過 WebP**
2. 第二候補: **透過 PNG**

## 推奨画像仕様

- 背景透過(単色背景を前提にしない。サイトの背景はページや将来のデザインで変わり得る)
- 正方形または縦長
- 長辺 **1600px 以上**(表示は最大 280px 四方だが、高解像度端末と将来の用途拡大に備える)
- キャラクター全体の周囲に十分な透明余白を残す
- 顔・耳・尻尾・手足を画像端へ寄せすぎない
- 文字・ロゴ・背景装飾を画像内へ埋め込まない(文字情報はサイト側で持つ)
- スマートフォンで 224px 程度に縮小しても顔が判別できる
- ファイル容量は可能な範囲で最適化する(WebP 品質 80 前後を目安)

## 透過余白の考え方

表示領域は正方形で、`object-fit: contain` 相当により**画像全体が必ず収まる**
(cover による顔・耳・尻尾の切断はしない)。つまり画像内の透明余白が
そのまま見た目の大きさを決めるため、次を目安にする。

- 全周に画像サイズの **5〜10% 程度**の透明マージン
- ヒーローでは浮遊アニメーション(上下 ±14px)があるが、これは transform に
  よる移動でコンテナ基準のため、画像自体に追加の余白は不要
- 余白を入れすぎるとキャラクターが小さく見えるので、上記レンジに収める

## 配置場所と想定ファイル名

```text
public/images/supitaro/supitaro-main.webp     ← ヒーロー用
public/images/supitaro/supitaro-profile.webp  ← プロフィール用
```

画像は原則 `public/` 配下のローカル管理とし、外部画像ドメインは追加しない。

## hero 画像の差し替え手順

1. 画像を `public/images/supitaro/supitaro-main.webp` に置く
2. `config/site.ts` の `images.hero` を設定する

```ts
hero: {
  src: "/images/supitaro/supitaro-main.webp",
  alt: "すぴたろう", // 画像の内容を表す代替テキスト(必須)
},
```

これだけで `FloatingSupitaro` がプレースホルダーから `next/image` 表示に切り替わる。
ヒーローの主要画像のため **priority(先行読み込み)は設定済み**。
表示は 224px(〜639px 幅)/ 280px(640px〜)の正方形領域 + `object-contain`。
`sizes` 設定済みのため、モバイルで不要に巨大な画像は読み込まれない。
プレースホルダーと同寸の領域に入るため **layout shift は発生しない**。

## profile 画像の差し替え手順

1. 画像を `public/images/supitaro/supitaro-profile.webp` に置く
2. `config/site.ts` の `images.profile` を同様に設定する

プロフィール画像は**通常の遅延読み込み**(priority なし)。表示は 224px 正方形 + `object-contain`。

## alt の設定方法

- `config/site.ts` の各 `SiteImage.alt` に、画像の内容を表す短い日本語を記入する
- alt が空文字の場合はキャラクター名(すぴたろう)でフォールバックされるが、
  実画像設定時は必ず明示する
- プレースホルダー(未設定時)は `role="img"` + aria-label「〜(画像準備中)」を維持

## Open Graph 画像は別途必要

ヒーロー画像の流用は不可。OG 用には**背景を焼き込んだ 1200×630px**(透過不可)を
別途用意し、`config/site.ts` の `images.og` に設定のうえ `app/layout.tsx` の
`openGraph.images` に配線する(TODO コメント箇所)。あわせて `siteUrl`(本番ドメイン)の
設定が必要。favicon は `app/icon.svg` の差し替えで管理する。

## モバイル確認 viewport

優先(モバイルファースト): **360×800 / 390×844**
追加確認: 768×1024 / 1024×768 / 1440×900

## 画像追加後に必要なブラウザ監査項目

- [ ] 全 viewport × `/` `/profile` で横スクロールなし
- [ ] 顔・耳・尻尾が切れていない(object-contain の確認)
- [ ] 画像とテキストが重ならない
- [ ] 404 リソース・壊れた画像がない
- [ ] layout shift が発生しない(プレースホルダーとの寸法一致)
- [ ] 浮遊アニメーションが画像でも過剰でない・コンテナ外へ大きくはみ出さない
- [ ] `prefers-reduced-motion` で浮遊停止
- [ ] スマートフォン実機相当(360px)で顔が判別できる
- [ ] alt が適切に設定されている(スクリーンリーダー読み上げ)
- [ ] Console エラー・hydration warning なし
- [ ] 転送サイズが過大でない(WebP 最適化の確認)
