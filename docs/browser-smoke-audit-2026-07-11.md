## 発見した問題と修正

### 問題 1(必須修正): JavaScript 無効時に FadeIn 配下のコンテンツが非表示

- **原因**: 旧 FadeIn は Motion の `initial={{opacity:0}}` を使っており、SSR の HTML にインライン `opacity:0` が出力される。JS が実行されないと `whileInView` が発火せず、トップページのキャラ紹介・チャンネル・お知らせが不可視のままになる
- **修正**: `components/FadeIn.tsx` を書き換え。サーバー出力は常に表示状態とし、JS 有効時のみマウント後(hydration 完了後)に画面外のセクションへ `opacity-0 translate-y-4` を付与、IntersectionObserver + CSS transition(500ms)でフェードイン。初期表示時点で画面内にある要素は隠さない(ちらつき防止)。`prefers-reduced-motion` 時は何もしない(常時表示)。状態変更は hydration 後のため mismatch なし
- **検証**: JS 無効で 3 ページとも全セクション opacity 1・本文/ナビ/リンク閲覧可。JS 有効でスクロールすると `opacity [1,0,0] → [1,1,1]` と正しくフェード。コンソールエラーなし

### 問題 2(監査で発見): コントラスト不足(補助テキスト 4 種)

WCAG AA(通常テキスト 4.5:1)未達を実測:

| 箇所 | 実測 | 修正 | 修正後 |
|---|---|---|---|
| チャンネルカード「準備中」バッジ(lilac-600 on lilac-200) | 3.88 | 文字色を deepblue-700 に | 約 8.7 |
| プレースホルダー「画像準備中」(deepblue-600/70) | 4.14–4.28 | 不透明度指定を外し deepblue-600 に | 約 9.7 |
| 無効チャンネルカード説明文(night-800/60) | 4.41 | /60 → /70 | 6.14 |
| 「準備中です…」等の空状態文(night-800/60) | 4.40 | /60 → /70 | 6.12 |

修正後の全ページ実測ワーストは 4.9(16px bold「YouTube 準備中」)で AA 合格。

### 問題 3(監査で発見): /news の見出し階層スキップ(h1 → h3)

ContentCard の見出しが h3 固定で、/news では h1 直下に h3 が来ていた。
`headingLevel` prop("h2" | "h3"、既定 "h3")を追加し、/news では h2 を指定。トップページは h2「お知らせ」配下の h3 のまま。視覚変化なし。

## 表示確認(§5)結果 — 全 15 組み合わせ

- 横スクロール: **なし**(`scrollWidth <= clientWidth` を全組み合わせで確認)
- テキスト切れ・画像との重なり: なし(スクリーンショット目視: 360 全 3 ページ、スクロール後トップ 360/1440、プロフィール 360、ニュース 360、No-JS トップ)
- ヘッダー崩れ: なし。フッター: 全ページで文書末尾に密着(gap 0–1px)、短いページでも不自然に浮かない

- PC 幅: コンテンツは max-w-5xl で制約され広がりすぎない
- 備考: フルページスクリーンショット中央に sticky ヘッダーが写り込むのは Playwright の合成アーティファクトで、実表示の問題ではない
